A Brief History Of Data Science

The term "Data Scientist" was coined by Jeff Hammerbatcher and DJ Patil circa 2008.

Jeff was leading the first data team at Facebook then, and DJ Patil was leading the data team at Linkedin.

A Brief History Of Data Science

"Around 4Q05 they decided to establish a 'reporting and analytics' function that was more like traditional DW/BI, and to hire a 'research scientist' to do things like identify and evaluate algorithms for news feed ranking"

"I was hired in 1Q06 with the same title 'research scientist', but my role quickly evolved into supporting the functions of the reporting and analytics group. Some time in 3Q06, Adam D'Angelo returned and we discussed changing the two groups to be focused on 'Data Infrastructure' and 'Data Analytics', and in 4Q06 (I think) we merged them into the 'Data Team'."

Building Data Pipelines

建立資料流程

A data pipeline is nothing but a series of operations streamed together, which helps us to automatically capture, munge, and aggregate data so it can be used on a recurring basis.

Remember This Picture!

Building ETL Job

Many open-source tools out there: Luigi (Spotify), Pinball (Pinterest), Airflow (Airbnb) ...etc, or just use plain old Cron

A Real Life Example

An ETL Job is often represented as a DAG (Directed Acyclic Graph), the graph consists of tasks/nodes, and each task perform a specific set of data operations

Each task executes a specific set of operations

The end result is that we have a reliable batch job that computes the same metrics we want everyday

Why Bother Building Data Pipelines?

Jobs can now be run automatically, the data produced can be used to power dashboards so more users can consume your data/results. (自動化)

A subtle but more important point, this is a great learning process to pick up engineering best practices such as Don't repeat yourself, Modularization, Abstractions. (學習 Software Engineering)

It provides the foundation for building more specialized pipelines such as Machine Learning or A/B testing.

建立資料流程所需要的能力

1. Version control - the most popular by far is Git

2. Learn how to do code reviews and provide feedbacks effectively

3. Know how to test, dry-run, and debug when job fails

4. Dependency management, Scheduling, Resource Allocation, Monitoring, Error Reporting, Alerting

Experimentation

跑實驗

The ability to run experiments breaks a lot of the limitation in traditional social science research, where techniques such as causal inference are heavily leveraged when RCE is not possible.

The ability to run massive experiment to improve product and decision making is, in my opinion, one of the most scientific aspects of our work.

跑實驗所需要的能力

Hypothesis Testing: Statistical test, p-values, statistical significance, power, effect size, multiple testing

Pitfalls of Experimentation: Carryover effect, metrics cherry-picking, data dilution, bucket anomaly

When a randomized controlled experiment is not possible, use Causal Inference techniques - difference in difference, instrumental variable, regression discontinuity design, market level experiments ...etc
